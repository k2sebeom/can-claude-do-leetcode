# Instruction

Read problem under ./dist/problems/ and write your solution under ./dist/solutions in a markdown format.

# Solution Format

Write in .md file with the same filename as the problem.

Solution document should be:

**Code**: in code block
**Chain of Thought**: Imagine you are a test applicant seeing this question for the first time. Mimic the chain of thought in finding the answer to this code challenge. Make it like a real process that applicant solves during the interview. For every decision you make, every insight you follow, lay out detailed explanation and reason. If applicable, write the code that should be written at the step.
**Complexity**: Details on time complexity of your solution.
**Algorithms**: Notable algorithms that are related.
